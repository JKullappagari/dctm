Password Expiry

 

Password expiry policy forces users to change their passwords once in every 90 days (configurable parameter).

 

From 10 days (configurable parameter) before the password expiry date, user will get a warning message stating password will expire in n days (based on the no of days left).

 

These settings defined in the <appSettings> section of the Web.config file determine the password expiry days and warning days:

1.     PwdExpiryDays – specified in number of days after which the user is forced to change the password. This is a global parameter and applicable for all the users of the DCTrackMobileWeb system.

2.     ExpirationWarningDays – specified in number of days. This determines the number of days before the password expires for the user, a message will appear each time the user logs in to the system.
