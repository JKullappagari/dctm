The DCTrackMobileWeb System

 

System Overview

 

The DCTrackMobile is a mobile solution that tracks assets, specifically in a Data Center environment. It allows for synchronization of data to provide asset and media’s location awareness. The specific functions related to the mobile application are described in the DCTrackMobile User Guide document. The DCTrackMobile also has a Web Component (DCTrackMobileWeb) which is primarily used for the following purposes:

1.     Import data from excel spreadsheet to DCTrackMobile Database.

2.     Maintain Master Data such as sites, locations, asset types, Manufacturer, Model etc. A full list of Master data components is given later in this document.

3.     Maintain and create new assets, searching assets based upon various parameters.

4.     Import Asset from excel spreadsheet to quickly populate the asset data instead of creating them one by one.

5.     Performing various transactions related to an asset:

a.     Asset Movement Restrictions for a period or permanently

b.     Decommissioning asset

c.     Write-off an asset

d.     Inventory reconciliation.

6.     Define and maintain application and asset relationship:

a.     Create Application

b.     Define application parameters like application type, criticality, associate a division (department) to application.

c.     Create and maintain mapping of application and assets.

d.     Import applications to quickly populate the application related data and mapping of application to asset

7.     User rights management to specify who can do what functions of the system.

8.     Generate Reports.

DCTrackMobile empowers users with the data they need to make real time decisions allowing them to inventory assets, move assets, commission new assets, or remove assets. Additionally, they can also manage the application and asset relationship to have a complete dependency of the asset.
