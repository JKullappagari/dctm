Import Asset

 

This section describes about how to import an asset from an excel spreadsheet.

 

To import Asset,

8.      Navigate Asset>>Import Asset on the DCTrackMobileWeb menu bar.

You will see the Import Asset Screen.

Figure 133: Import Asset Screen

9.      Enter Company to which Assets will be added in the required field and Click Upload File  button to select the Asset file (excel file).

10.   Multiple files can be selected using Upload File option. Use Add button to add multiple files, also individual files can be removed using cross (X) next to the file.

11.   After selecting asset file(s) click on Upload  button to parse the file.

Figure 134: Import Asset - Multiple File selection

12.   Upload option will upload and parse all the selected files and shows combined file statistics.

Figure 135: Import Asset Screen after parse

 

 

13.   Once file is successfully parsed it will show the statistics in terms of how many Asset, unique sites, rooms, rows and racks, manufacturers, models and asset types found in the excel file.

14.   Click Import to DB  button to import the data into the DCTrackMobileWeb database.

Figure 136: final Import Asset Screen

 

15.   On successful import it will show the statistics of how many assets imported successfully and how many failed. For the records that has one or more errors, the detail of the error is not shown in this page. Instead only a ‘Success/Failure’ status is shown under the Reason column of the grid. This data can be exported to excel using excel by using the standard export functionality. This will be useful to view the details of the error messages, correct the input records and then import them again.

Can I modify column names in the import excel file?

No. Column names should be same as mentioned above. Even keep the excel sheet name as Assets. If no data passed for mandatory columns than those records will fail during import.

              

 

 

What will happen if I import the same asset again?

If an asset has already been added and the same asset exists in the excel file, then this asset will again be added to the database unless SerialNo-Model check is enabled. If this option is enabled than system will allow only one asset with same serial no and model name.

 

We recommend to follow the process, mentioned below, to import the assets to the DCTrackMobile’ s database.

 

The process starts with initial import of the assets. Assuming there are few records that failed the validation, we recommend that you identify the cause of the error (most common causes are ‘Room not defined, Asset type not defined etc.) and rectify accordingly.

 

Copy the rectified rows into a separate spreadsheet and import it again to using the Import Asset function.

 

Repeat the process till all your desired assets are imported to the database.

 

 

Update in v3.5:  DCTrackMobileWeb now supports duplicate elimination during asset import based on Serial No and Model combination. This feature can be set off by changing ImportAssetUniqueFilter value to 0 in web.config file.

          ImportAssetUniqueFilter = 0 (no filter)

          ImportAssetUniqueFilter = 1 (Serial No + Asset Tag)

How it works:

          When 2 assets with same serial no (SGH123) and/or same Asset tag (Tag123) imported than first one will be imported successfully and second one will fail with a message “duplicate asset with same SerialNo and/or AssetTag”

          This feature can be disabled by setting the value to 0 in web.config.
