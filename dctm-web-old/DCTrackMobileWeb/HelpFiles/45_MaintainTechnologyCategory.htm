Maintain Technology Category

 

The Technology Category screen will enable privileged users to define and maintain different Technology Categories list. Technology category is a broad category to group assets such as Storage, Network, Mainframe, UNIX, etc.  This is another asset classification and used for different reporting purposes. Please note that this field is not mandatory.

 

Can I create a Technology Category of the same name?

No. Technology Category should be unique. System will inform you that the record already exists.

 

I have already categorized the assets by Asset Type field. Why do I need to categorize them again with Technology Category?

Typically, Asset Type defines an asset category. This definition depends on the user as to how granular they want to define the asset type. Such as one can define a very broad Asset Type such as Server or Network and create all the corresponding assets, whereas one can also create granular asset types such as ‘Hub’, ’24 Port Switch’ etc. and create the corresponding assets. While any approach is fine, we recommend to create Asset Type as detailed as possible such that it gives a meaningful category of the asset. The technology category is a much higher-level definition such as Storage, Wintel Systems etc., for the assets to be categorized. The main purpose of the Technology Category is to enable the growth planning for each of the technology category for future period. But this topic is not covered under HP DCTrackMobile application.
